Create a full-featured DataTable component using a simple column definition array API (per CONTEXT.md: architecture is Claude's discretion; simple props API is the recommendation from RESEARCH.md).

**Type definitions:**

```typescript
export interface Column<T> {
  key: string;
  header: string;
  render?: (value: unknown, row: T) => ReactNode;
  sortable?: boolean;
  width?: string;       // Tailwind width class like 'w-1/4' or 'w-[200px]'
  align?: 'left' | 'center' | 'right';
}

type SortDirection = 'asc' | 'desc' | null;

interface SortState {
  key: string;
  direction: SortDirection;
}

interface DataTableProps<T> {
  columns: Column<T>[];
  data: T[];
  keyExtractor: (row: T) => string | number;
  onRowClick?: (row: T) => void;
  sortState?: SortState;
  onSort?: (state: SortState) => void;
  emptyIcon?: ElementType;
  emptyMessage?: string;
  className?: string;
}
```

**Implementation details:**

1. **Table structure:** Use `table-fixed w-full` for controlled column widths at 1366px. This prevents auto-layout from causing horizontal overflow.

2. **Header row:** For each column:
   - Render header text in `text-xs-causa font-semibold text-[var(--color-text-muted)] uppercase tracking-wider`
   - If `sortable` is true, render as a clickable header with sort indicator (ArrowUpDown, ArrowUp, ArrowDown from lucide-react, size 14)
   - Clicking a sortable header: if currently sorted asc on this column -> desc, if desc -> null (unsorted), if null/other column -> asc on this column
   - Header cell padding: `px-4 py-3`
   - Apply `text-left/center/right` based on column.align (default: left)

3. **Data rows:** For each row:
   - `border-b border-[var(--color-border)] last:border-0`
   - Zebra striping: `even:bg-[var(--color-surface-alt)]/50` (subtle, not jarring)
   - If `onRowClick` is provided: add `hover:bg-[var(--color-surface-alt)] cursor-pointer transition-causa` + `focus-visible:outline-none focus-causa` + `tabIndex={0}` + Enter/Space key handler
   - Cell padding: `px-4 py-3 text-sm-causa text-[var(--color-text)]`
   - If column has `render`, call `render(row[column.key], row)` — otherwise render `String(row[column.key])`
   - Apply column width class if specified

4. **Sorting:** DataTable is a controlled component for sort. Parent manages sort state and passes sorted data. DataTable just renders what it receives and calls `onSort` when headers are clicked. This keeps the component simple and lets Phase 3 pages handle their own sort logic (server-side or client-side).

5. **Empty state:** When `data.length === 0`, render EmptyState component (import from `./empty-state`) with colSpan equal to columns.length. Use `emptyIcon` (default: Inbox from lucide-react) and `emptyMessage` (default: "Nenhum registro encontrado").

6. **Responsive:** With `table-fixed`, column widths are controlled. For 1366x768 with sidebar (240px), content area is ~1126px. With 6 columns, each averages ~188px. Use `min-w-0 truncate` on cell content to prevent overflow. Columns without explicit width distribute evenly.

7. **No hardcoded hex values** — all colors via var() tokens.

Export `DataTable` (generic component) and `Column` type.
