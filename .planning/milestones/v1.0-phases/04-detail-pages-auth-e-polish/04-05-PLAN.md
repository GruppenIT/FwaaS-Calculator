**A11Y-01 — WCAG AA Contrast Audit:**

1. Read globals.css fully — extract all color token values from `:root` (light) and `.dark` sections.

2. Calculate contrast ratios for these critical pairings:
   - `--color-text` on `--color-bg` (body text)
   - `--color-text` on `--color-surface` (card text)
   - `--color-text-muted` on `--color-bg` (muted text on background)
   - `--color-text-muted` on `--color-surface` (muted text on cards)
   - `--color-text-muted` on `--color-surface-alt` (muted text on alt surfaces)
   - `--color-primary` on `--color-bg` (primary interactive text/links)
   - `--color-primary` on `--color-surface` (primary on cards)
   - Badge text colors on their backgrounds (success, warning, danger variants)

   Do this for BOTH `:root` (light) and `.dark` themes.

3. KNOWN RISK from RESEARCH.md: `--color-text-muted: #9ca3af` on dark surface `#1a1a2e` gives approximately 4.15:1 — FAILS AA (needs 4.5:1 for normal text at 11-13px). Fix by bumping `--color-text-muted` in `.dark` to approximately `#b3b8c4` or higher to achieve at least 4.5:1 against the darkest surface it appears on.

4. For each failing pair, adjust the token value in `.dark` or `:root` to meet the threshold:
   - Normal text (under 18px / under 14px bold): 4.5:1 minimum
   - Large text (18px+ / 14px+ bold): 3:1 minimum

5. PRESERVE the visual identity — adjustments should be minimal. Do not change primary brand colors. Only adjust muted/secondary text colors if needed.

**A11Y-02 — Focus Ring Verification:**

1. Search the codebase for all interactive elements that should have focus rings:
   - Buttons: Should use `focus-causa` utility (verify in button.tsx)
   - Inputs: Should have focus styles (verify in input.tsx)
   - Links/NavLinks: In sidebar, breadcrumbs
   - Table rows: DataTable rows (verify `focus-causa` on `<tr>`)
   - Tab buttons: New tab bar in processo-detail-page.tsx (Plan 01 should have added `focus-causa`)

2. If any interactive element is missing `focus-causa` class or equivalent focus styling, add it. The `focus-causa` utility is defined in globals.css and provides a consistent focus ring.

3. Verify that the `focus-causa` utility itself produces a visible ring in both themes. If needed, ensure the focus ring color has sufficient contrast against both light and dark backgrounds.

4. Check the new tab buttons from Plan 01 have `focus-causa` applied.

5. Check that the new login page form elements from Plan 02 have focus styles.
