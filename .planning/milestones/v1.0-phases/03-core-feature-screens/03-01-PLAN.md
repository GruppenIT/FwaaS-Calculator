    2. Export `useChartTheme()` hook:
       - Uses `useState` with a tick counter and `useEffect` with `MutationObserver` on `document.documentElement` watching `{ attributes: true, attributeFilter: ['class'] }`. On mutation, increment tick to force useMemo recomputation. Disconnect observer on cleanup.
       - `useMemo` depends on `[tick]`. Inside, calls `resolveCssVar` for each token:
         - colors: primary (--color-primary), warning (--color-warning), success (--color-success), danger (--color-danger), muted (--color-text-muted), amber (--color-accent-amber), emerald (--color-accent-emerald)
         - gridProps: `{ strokeDasharray: '3 3', stroke: border }` where border = resolveCssVar('--color-border')
         - axisProps: `{ tick: { fontSize: 11, fill: muted } }`
         - tooltipProps: `{ contentStyle: { background: surface, border: '1px solid ${border}', borderRadius: '6px', fontSize: 12, color: text } }` where surface = resolveCssVar('--color-surface'), text = resolveCssVar('--color-text')
       - Return `{ colors, gridProps, axisProps, tooltipProps }`

    Create test file `use-chart-theme.test.ts`:
    - Mock `getComputedStyle` to return known values for each CSS var. Mock `document.documentElement`.
    - Test `resolveCssVar` directly (pure function).
    - For hook tests, since vitest environment is 'node', test the exported `resolveCssVar` function only. The hook structure (MutationObserver + useMemo) is verified via the dashboard visual checkpoint.

    IMPORTANT: Do NOT use `var()` strings in the return values. Every color must be a resolved hex/rgb string.
