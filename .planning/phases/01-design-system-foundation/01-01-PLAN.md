<task type="auto">
  <name>Task 3: Add ESLint no-hardcoded-hex custom rule</name>
  <files>eslint.config.js</files>
  <action>
Read the existing `eslint.config.js` and add a custom inline plugin that flags hardcoded hex color values (#xxx through #xxxxxxxx) in string literals within files under `components/ui/` and `pages/`.

Add this plugin object at the top of the file (before the export):

```javascript
const noHardcodedHexPlugin = {
  meta: { name: 'causa-custom-rules' },
  rules: {
    'no-hardcoded-hex': {
      meta: {
        type: 'suggestion',
        docs: { description: 'Disallow hardcoded hex colors in UI components — use CSS custom properties instead' },
        schema: [],
      },
      create(context) {
        return {
          Literal(node) {
            if (typeof node.value !== 'string') return;
            if (!/#[0-9a-fA-F]{3,8}\b/.test(node.value)) return;
            const filename = context.filename || context.getFilename();
            if (filename.includes('components/ui/') || filename.includes('pages/')) {
              context.report({
                node,
                message: 'Hardcoded hex color "{{value}}" found. Use CSS custom properties (var(--color-*)) or Tailwind causa-* utilities instead.',
                data: { value: node.value },
              });
            }
          },
        };
      },
    },
  },
};
```

Then add to the exported config array a new entry:

```javascript
{
  plugins: { 'causa': noHardcodedHexPlugin },
  rules: { 'causa/no-hardcoded-hex': 'error' },
}
```

This rule only fires on files in `components/ui/` and `pages/` — it will NOT flag hex values in globals.css (which is correct since that is where hex values belong).
