   ```typescript
   const styleMap: Record<ToastType, string> = {
     success: 'border-[var(--color-success)]/30 bg-[var(--color-success)]/8 text-[var(--color-success)]',
   ```

**Skeleton (skeleton.tsx):**
1. The Skeleton component uses inline `style={{ animation: 'skeletonPulse 1.5s ease-in-out infinite' }}`. The globals.css prefers-reduced-motion guard from Plan 01 handles this globally via `animation-duration: 0.01ms !important`. However, because the animation is set via inline style, the `!important` in the media query may not override it in all browsers.

   Fix: Replace the inline style animation with a Tailwind arbitrary animation class:
   ```tsx
   className={`bg-[var(--color-surface-alt)] rounded-[var(--radius-md)] animate-[skeletonPulse_1.5s_ease-in-out_infinite] ${className}`}
   ```
   Remove the `style` prop entirely. This allows the CSS `animation-duration: 0.01ms !important` from the media query to properly override.

2. Update the background token from `bg-causa-surface-alt` to `bg-[var(--color-surface-alt)]` for consistency with other components.

3. Keep SkeletonText and SkeletonTableRows exports unchanged — they delegate to Skeleton.
