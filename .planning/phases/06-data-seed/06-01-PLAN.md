  <action>
    Create `packages/database/src/seeds/seed-demo.ts` that:

    **Setup:**
    - Import `@faker-js/faker` with pt_BR locale (`import { fakerPT_BR as faker } from '@faker-js/faker'`)
    - Import `createDatabase` from `../client.js` and ALL schema tables from `../schema/index.js`
    - Import `v4 as uuid` from `uuid`
    - Create database with `{ topologia: 'solo', sqlitePath: 'causa-dev.db' }`
    - Set faker seed to a fixed value (`faker.seed(42)`) for deterministic output

    **Idempotent clear (SEED-04):**
    - Before inserting, DELETE all rows from tables in reverse-dependency order to avoid FK violations:
      `timesheets, parcelas, despesas, documentos, agenda, tarefas, movimentacoes, prazos, honorarios, processos, contatos, clientes`
    - Do NOT delete from `users`, `roles`, `permissions`, `role_permissions` — those come from the RBAC seed
    - Use `db.delete(tableName).run()` for each table
    - Log: `console.log('Limpando dados demo existentes...')`

    **Data generation quantities:**
    - 15 clientes (mix of PF and PJ, ~60% PF / 40% PJ)
    - 25 processos (linked to random clientes)
    - 8 contatos (various types: perito, correspondente, testemunha, etc.)
    - ~50 movimentacoes (2-3 per processo, distributed over 12 months)
    - ~30 prazos (see SEED-03 requirements below for urgency distribution)
    - 15 honorarios with ~30 parcelas
    - 20 tarefas
    - 15 agenda events
    - 10 documentos
    - 12 despesas
    - 40 timesheets

    **Temporal distribution (SEED-02):**
    - Use a helper function `randomDateInLastMonths(months: number): string` that generates ISO date strings distributed across the last N months
    - Processos: `dataDistribuicao` spread across last 12 months
    - Movimentacoes: `dataMovimento` spread across last 12 months, always AFTER the processo's dataDistribuicao
    - Tarefas: `createdAt` spread across last 6 months
    - Timesheets: `data` spread across last 2 months (for produtividade chart)
    - Agenda events: mix of past (60%) and future (40%) events
    - Parcelas: `vencimento` spread across last 6 months with some in the future

    **Urgency tier prazos (SEED-03):**
    Among the ~30 prazos, ensure these SPECIFIC prazos exist with status='pendente':
    - 1 prazo with `dataFatal` = today or tomorrow (fatal tier, 0-1 days)
    - 1 prazo with `dataFatal` = yesterday (fatal + vencido)
    - 3 prazos with `dataFatal` = 2-3 days from now (urgente tier)
    - 3 prazos with `dataFatal` = 4-7 days from now (semana tier)
    - 5+ prazos with `dataFatal` = 8+ days from now (proximo tier)
    - Remaining prazos can have status='cumprido' with dates in the past

    **Realistic Brazilian legal data:**
    - CNJ numbers: format `NNNNNNN-NN.NNNN.N.NN.NNNN` (7 digits, 2 check, 4 year, 1 justice, 2 state, 4 court)
    - Clientes PF: Brazilian names via faker.person, CPF via `faker.string.numeric(11)` formatted as `NNN.NNN.NNN-NN`
    - Clientes PJ: Company names via faker.company, CNPJ via `faker.string.numeric(14)` formatted as `NN.NNN.NNN/NNNN-NN`
    - Tribunais: pick from `['TJSP', 'TJRJ', 'TJMG', 'TJRS', 'TJPR', 'TRT2', 'TRT15', 'TRF3']`
    - Comarcas: pick from `['Sao Paulo', 'Rio de Janeiro', 'Belo Horizonte', 'Curitiba', 'Porto Alegre', 'Campinas']`
    - Varas: pick from `['1a Vara Civel', '2a Vara Civel', '3a Vara do Trabalho', 'Vara de Familia', '1a Vara Federal']`
    - Honorario valores: range R$2.000 to R$50.000
    - Parcela valores: divide honorario by numeroParcelas
    - Despesa valores: range R$50 to R$5.000

    **User references:**
    - Query existing users from the database (created by RBAC seed) and use their IDs for all `userId`, `criadoPor`, `responsavelId`, `advogadoResponsavelId` references
    - If no users exist, create 4 demo users with roles: admin, advogado, estagiario, secretaria (query roles table to get roleIds)

    **Insert order (respecting FK constraints):**
    1. Users (if needed) -> query existing users
    2. Contatos (no FK deps)
    3. Clientes
    4. Processos (needs clientes + users)
    5. Movimentacoes (needs processos)
    6. Prazos (needs processos + users)
    7. Honorarios (needs processos + clientes)
    8. Parcelas (needs honorarios)
    9. Tarefas (needs processos + users)
    10. Agenda (needs processos + users)
    11. Documentos (needs processos + users)
    12. Despesas (needs processos + users)
    13. Timesheets (needs users + processos)

    **Logging:** After each entity type, log count: `console.log('  OK {N} {entity} criados')`
    At the end: `console.log('Seed demo concluido com sucesso!')`

    **Important implementation notes:**
    - Use synchronous `.run()` calls (same pattern as existing run.ts) since this is SQLite
    - Generate all UUIDs with `uuid()` (v4)
    - Store all generated IDs in arrays for cross-referencing (e.g., `const clienteIds: string[]`, `const processoIds: string[]`)
    - For documentos, use fake hashes (`faker.string.hexadecimal({ length: 64 })`) and set `tamanhoBytes` to random values
    - For timesheets, calculate `valorCalculado` from `duracaoMinutos` and a taxa horaria
